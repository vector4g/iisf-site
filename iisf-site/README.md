## SEO Automation

### 1. Auto-generated sitemap

The sitemap is available at:

`https://intersectionalsafety.org/sitemap.xml`

It is generated by `app/sitemap.ts` and includes:
- Static pages
- Blog posts from Sanity
- Fallback to local MDX posts if Sanity is unavailable

### 2. Sanity publish webhook -> revalidate + IndexNow

Webhook endpoint:

`POST /api/seo/webhook`

When a post is created/updated/deleted, this endpoint:
- Revalidates affected routes (`/`, `/blog`, `/blog/[slug]`, feeds, sitemap)
- Submits changed URLs to IndexNow (if configured)

Required environment variables:

```env
SANITY_WEBHOOK_SECRET=your-random-shared-secret
INDEXNOW_API_KEY=your-indexnow-key
NEXT_PUBLIC_SITE_URL=https://intersectionalsafety.org
# Optional override:
# INDEXNOW_KEY_LOCATION=https://intersectionalsafety.org/<key>.txt
```

Sanity webhook recommendations:
- URL: `https://intersectionalsafety.org/api/seo/webhook`
- Method: `POST`
- Header: `Authorization: Bearer <SANITY_WEBHOOK_SECRET>`
- Filter: `_type == "post"`
- Projection:
  ```groq
  {
    _type,
    operation,
    "slug": slug.current
  }
  ```

### 3. CI SEO lint

Run locally:

```bash
npm run seo:lint
```

GitHub Actions workflow:

`.github/workflows/seo.yml`

It runs SEO lint on relevant SEO/content changes.
